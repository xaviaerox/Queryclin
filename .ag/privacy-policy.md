# Privacy Policy - Queryclin

## 1. Philosophy: Privacy-First & Local-First
Queryclin is built on the principle of **Absolute Clinical Privacy**.
- **Local-First:** All clinical data processing occurs strictly within the user's browser/local environment.
- **Zero Cloud Persistence:** Queryclin does not store, transmit, or persist clinical data to any cloud service or external database.
- **Local Processing:** The system is a static tool that operates on local files provided by the user at runtime.

## 2. Data Protection
The repository must remain a "clean zone". 
- **Absolute Anonymization:** Only synthetic (mock) datasets are permitted within the repository for testing and demonstration.
- **Separation of Concerns:** There is a strict boundary between the **application code** (public) and the **clinical data** (private/local).

## 3. User Responsibility
Users are informed that:
- They are responsible for the clinical data they load into their local instance of Queryclin.
- The public deployment (GitHub Pages) is a static engine; it does not "see" or "receive" the data loaded by the user locally.

## 4. Compliance
Documentation and development must align with GDPR/HIPAA-level privacy standards regarding the *handling* (or rather, the non-handling) of clinical information.
