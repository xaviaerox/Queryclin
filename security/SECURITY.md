# Security Policy - Queryclin

## 1. Security Philosophy
Queryclin is a **privacy-first** and **local-first** application. Our security model relies on the fact that clinical data never leaves the user's local environment.

## 2. Reporting a Vulnerability
We take security seriously. If you discover a security vulnerability, please do NOT open a public issue. Instead, follow these steps:
1. Email the maintainer at [maintainer-email@example.com] (Replace with actual contact if available).
2. Provide a detailed description of the vulnerability and steps to reproduce it.
3. We will acknowledge your report within 48 hours and provide a timeline for resolution.

## 3. Scope
This policy covers the Queryclin source code and its official public deployment. It does NOT cover:
- Data breaches resulting from the user's local machine being compromised.
- Vulnerabilities in 3rd party dependencies (though we strive to keep them updated).

## 4. Contributing Safely
All contributors must adhere to the [Safe Contributing Guide](./SAFE_CONTRIBUTING.md). Any PR containing sensitive data will be rejected and the history purged.
